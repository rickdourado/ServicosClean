## PROMPT ALTERNATIVO - SCRIPT DE INFORMAÇÃO

Ajuste o texto fornecido para o formato de SCRIPT DE INFORMAÇÃO da Prefeitura do Rio de Janeiro.
Não crie, complete, interprete nem acrescente informações: utilize exclusivamente o conteúdo enviado como base, apenas reorganizando e reescrevendo.

Contexto:
Diferente dos Scripts de Serviço (focados em ação/solicitação), os Scripts de Informação focam na compreensão, orientação e contextulização. O cidadão lê para entender, não para pedir.

Estrutura obrigatória:
Utilize exatamente os seguintes tópicos, nesta ordem e retorne um JSON com estas chaves:

- `o_que_e`: O que é (Definição clara do tema/política/equipamento).
- `como_funciona`: Como funciona (Funcionamento, regras gerais, aplicação prática na vida do cidadão).
- `publico_alvo`: Público-alvo (Para quem a informação é relevante. Não use como critério de elegibilidade/restrição, mas como direcionamento).
- `informacoes_importantes`: Informações importantes (Contexto, observações relevantes, dados complementares).

Regras de escrita:
1. Linguagem simples, clara e cidadã. Evite termos técnicos, jurídicos ou burocráticos.
2. Preserve integralmente o sentido e as ideias originais.
3. Não use linguagem de solicitação, pedido ou atendimento (ex: "solicite", "agende").
4. Não utilize os tópicos “Para que serve” ou “Quem pode solicitar”.
5. Não repita informações entre os tópicos.
6. Em "Público-alvo", deixe claro que o acesso é aberto/informativo, sem criar barreiras de solicitação.

Objetivo:
Traduzir o conteúdo fornecido para um formato informativo padronizado, permitindo que o cidadão compreenda o tema de forma rápida e acessível, eliminando a lógica de "serviço solicitado".
